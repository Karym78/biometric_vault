Course Security Model

DICAN and biometric protection

This page maps the security principles from the course to the concrete controls implemented in the application.

Principle Course idea Implementation in this app
Confidentiality Protect sensitive data from unauthorized reading. Files are encrypted with AES-256-GCM before storage.
Integrity Detect unauthorized modification. AES-GCM authenticates encrypted files, logs are chained with hashes, and fingerprints include SHA-256 protected templates.
Control access Only authorized users can reach protected resources. Session login is required, encrypted file selection is checked per user, and watermarked images are served only to their owner.
Authentication Verify identity using a biometric characteristic. Fingerprint upload simulates acquisition, then preprocessing, minutiae matching, score, threshold and decision are applied.
Non-repudiation / Traceability Keep evidence of important actions. Registration, login, encryption, decryption, attack simulations and watermark checks are stored in security logs.
Privacy / RGPD Biometric data must be justified, protected and transparent. Registration requires consent; raw images are not used as login credentials; the app explains the processing purpose.
